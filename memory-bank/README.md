# Memory Bank

Project context for Cursor AI — helps maintain persistent understanding across sessions.

## Files

| File | Purpose |
|------|---------|
| `projectbrief.md` | Core requirements, goals, constraints |
| `productContext.md` | Personas, journeys, functional scope |
| `systemPatterns.md` | Architecture, services, patterns |
| `techContext.md` | Stack, integrations, API design |
| `activeContext.md` | Current focus, in-progress decisions |
| `progress.md` | Status, completed work, observations |
| `tasks.md` | Task tracking, backlog |
| `creative/` | Design decision documents |
| `reflection/` | Post-implementation reviews |
| `archive/` | Archived task documentation |

## Usage

- **Cursor:** These files are indexed and available as context. Reference them when planning or implementing.
- **Update** `activeContext.md` when starting a new task; `progress.md` and `tasks.md` as you ship.
